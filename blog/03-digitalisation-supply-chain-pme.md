# Conseil Digitalisation Supply Chain : WMS, TMS, ERP pour PME

**"Quelle techno faut-il choisir?"**

C'est la question. Et c'est aussi le piège.

Après 12+ ans de projets digitaux (DHL, Renault, SOMACA, Diana Holding), je vois le même pattern : PME investissent dans WMS/TMS/ERP, mais sans stratégie. Résultat : Adoption 40-50%, ROI négatif.

**La technologie seule ne change rien.** C'est la stratégie + processus + changement qui crée valeur.

Voici comment faire juste.

---

## Les 5 Erreurs Tech qui Coûtent Cher

### Erreur #1 : Vous Choisissez avant d'Auditer

**Symptôme :** Vous trouvez un WMS "cool", vous l'implémentez, puis découvrez qu'il ne match pas vos processus.

**Coût :** Rebusquer WMS (50-100K MAD), re-implémentation 6 mois de retard.

**Bon ordre :**
1. Audit processus ACTUEL (2-3 jours, 3-5K MAD)
2. Design futur état (1-2 semaines)
3. Sélectionner tech qui FIT ce design
4. Implémenter

### Erreur #2 : Vous Ignorez Change Management

**Symptôme :** WMS go-live. Équipe continue Excel. Adoption 35%.

**Coût :** 60% du potentiel WMS gâché.

**Bonne approche :** 30% tech, 70% people.
- Communication claire
- Training robuste
- Quick wins visibles (semaine 1)
- Management sponsor actif

### Erreur #3 : Vous Sous-budgétez Implémentation

**Erreur budgétaire :** License WMS 50K → Vous croyez coût total 50K.

**Réalité :** 
- License : 20%
- Implementation : 50%
- Training : 15%
- Contingency : 15%

**Exemple :** WMS 50K license = Budget réel 250K MAD (50K÷20%).

### Erreur #4 : Vous Oubliez l'Intégration

**Symptôme :** WMS fonctionne, mais data ne sync pas avec ERP. Données qui se contredisent.

**Solution :** Planifier API/intégration DAY 1. Pas après go-live.

### Erreur #5 : Pas de Support Post-Go-Live

**Symptôme :** WMS go-live → Bugs → Pas de support → Users reviennent à Excel.

**Bonne approche :** Support 3 mois minimum post-go-live (1 jour/semaine minimum).

## Audit Tech Rapide — 3 Jours

Quand vous m'appelez, voilà ce qu'on fait:

**Jour 1: Cartographie État Actuel**
- Processus warehouse/transport : Actuel vs Documenté
- Data qualité : Stock accuracy, lead time, exceptions
- Tech stack : Utilisé vs Underutilized
- Équipe skills : Qui sait quoi faire

**Jour 2: Design État Futur**
- Processus optimisé (flows, cycles, décisions)
- Tech requirements : Fonctionnalités minimales
- Timeline réaliste
- Budget prévisible

**Jour 3: Recommandations**
- Quel système (WMS/TMS/ERP ou mix)
- Quel fournisseur (SAP/Infor/Prologix/Open-source)
- Roadmap 12 mois
- Coûts + ROI projection

## WMS : Quand & Comment

### Cas d'Usage #1 : PME Croissance (50-500 employés)

**Symptôme :** Warehouse 500-2000 m², Excel management, stock accuracy 82%, picking time 45 min/100 items.

**Solution :** WMS léger (Prologix, Netstock, ou custom).

**Timeline :**
- Audit + Design : 2 semaines
- Pilot zone (20% warehouse) : 4 semaines
- Rollout complet : 4-6 semaines
- Stabilisation : 8 semaines

**Total : 4-5 mois, coût 150-250K MAD**

**Résultats attendus :**
- Stock accuracy : 94%+
- Picking time : 22 min/100 items (-50%)
- Coûts opérationnels : -12-15%
- Équipe adoption : 85%+

**Cas réel — DHL Supply Chain (2010-2013) :**
- 3 plateformes, 21,000 m², 120 collaborateurs
- Baseline : Ruptures 22%, SLA 88%
- Post-WMS (6 mois) : Ruptures 12%, SLA 96%+
- Gain annual : 1.8 MMAD

### Cas d'Usage #2 : Warehouse Complex (2000+ m²)

**Symptôme :** Multi-zone, multi-client, complexité haute, Excel limite.

**Solution :** WMS complet (SAP, Infor, Kinaxis, RedPrairie — cher mais robuste).

**Timeline :** 9-14 mois

**Budget :** 500K-2 MMAD (license + implementation + training)

## TMS : Quand & Comment

### Cas d'Usage : Transport Optimization

**Symptôme :**
- Livraisons non-consolidées (1-2 items par shipment)
- Délai livraison moyen 3-4 jours
- Coût transport 25-30% du coût vente
- Traçage manual (email, call, WhatsApp)

**Solution :** TMS (MercuryGate, Fourkites, ou custom).

**Cas réel — Douja Promotion Addoha (2013-2014) :**
- Volume : 710 MMAD annuel, 31 chantiers Maroc/Afrique
- Transport partners : CMA-CGM, MAERSK, TNT
- Baseline : Shipments non-consolidé, cost +15% vs optimum
- Post-TMS (3 mois) : Consolidation 82%, cost -12%, delay reduced

**Timeline :** 3-4 mois

**Budget :** 100-250K MAD

**Résultats:**
- Consolidation ratio : +60%
- Cost per shipment : -15-20%
- Delay : -30%
- Visibility : Real-time (vs 2-3 jours)

## ERP Module Supply Chain : Quand l'Implémenter

### Cas d'Usage : PME avec Besoin Planning Intégré

**Symptôme :**
- Sales forecast ≠ Production plan ≠ Procurement
- Désynchronisation → Stock excessif ET ruptures
- ERP existe mais supply chain modules unutilized

**Solution :** Activation S&OP (Sales & Operations Planning) dans ERP.

**Cas réel — Diana Holding (2010) :**
- Groupe Boissons/Agro : ABC (Coca), Ebertec, Les Celliers, SNV
- Baseline : Stock 35% safety buffer, ruptures 8%, cash cycle 52 jours
- Post-S&OP (8 semaines) : Stock 18%, ruptures 2%, cash cycle 38 jours
- Saving : 2 MMAD cash freed

**Timeline :** 6-10 semaines (si ERP already existe)

**Budget :** 50-150K MAD (consulting + training)

## Implémentation : Phases Réalistes

### Phase 1: Audit & Design (2-4 semaines)

**Livrables :**
- Process flows (actuels vs futurs)
- Tech requirements (détaillées)
- Roadmap 12 mois
- Budget estimate +/- 20%

**Coût:** 5-15K MAD

### Phase 2: Selection & Configuration (4-8 semaines)

**Activités:**
- RFP envoyé à 3-5 vendors
- Demos & références check
- Contracting & license negotiation
- System configuration (base setup)

**Coût:** 20-50K MAD (consulting)

### Phase 3: Pilot Zone (4-8 semaines)

**Approche:**
- Sélectionner 20% de l'opération (1 warehouse zone, 1 transport partner)
- Run parallel (old + new system en même temps)
- Fix bugs, train users, collect feedback
- Go-live zone quand confidence 85%+

**Data points :**
- SOMACA iFA pilot (4 semaines) → Production +25% efficiency
- DHL WMS pilot (6 semaines) → Picking -30% time

**Coût:** Implementation resources (50-100K MAD)

### Phase 4: Rollout Complet (4-12 semaines)

**Timeline :**
- Semaine 1-2 : Setup new zones, parallel run
- Semaine 3-4 : Cut-over (stop old system, full on new)
- Semaine 5-8 : Stabilisation, bug fix, reporting
- Semaine 9-12 : Optimisation, retraining, documentation

**Coût:** Implementation + support (50-150K MAD)

### Phase 5: Stabilisation & Optimization (8-12 semaines)

**Focus :**
- KPI monitoring (adoption, quality, efficiency)
- User feedback loops
- Process tweaks
- Knowledge transfer

**Coût:** Ongoing support (20-40K MAD/mois)

## Sélectionner le Bon Vendor

### Critères de Sélection

1. **Fit fonctionnel** (50% poids)
   - Votre industrie/secteur (textile, agro, pharma, retail)
   - Size company (PME vs GE)
   - Complexity operations

2. **Support & Expertise Maroc** (30% poids)
   - Local support team
   - Références locales
   - Capacity accompagner transformation

3. **Coût Total Propriété** (20% poids)
   - License + implementation + training + support
   - Comparaison 3 ans, pas année 1

## Change Management : Les 70% Qui Décident du Succès

### User Adoption Challenge

**Data :** WMS sans change management → 40% adoption. Avec change management → 85% adoption.

**Approche :**
1. **Communication** (semaine 0-4)
   - Why (business case clair)
   - What (système, features)
   - When (timeline)
   - Who owns (executive sponsor visible)

2. **Training** (semaine 4-12)
   - Train-the-trainer approach
   - Hands-on 3 jours minimum
   - Job aids + quick reference guides
   - Post-go-live coaching

3. **Quick Wins** (semaine 1-4 post-go-live)
   - Picking time -30%? Communicate.
   - Stock accuracy +10%? Celebrate.
   - Users see benefit immediately → Adoption +40%

4. **Governance** (ongoing)
   - Weekly stand-ups (issues, resolutions)
   - Monthly KPI review
   - User steering committee (feedback direct)
